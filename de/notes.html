Berechtigung

Bedienungshilfen-Berechtigung

🔐

Warum wird die Bedienungshilfen-Berechtigung benötigt?

Workspace Container verwendet die macOS Accessibility API (AXUIElement), um Position und Größe anderer App-Fenster zu steuern. Dies ist der einzige legitime Weg zur Fensterverwaltung unter macOS — alle Fenstermanagement-Apps (wie Magnet, Rectangle) benötigen diese Berechtigung.

Workspace Container nutzt diese Berechtigung ausschließlich zum Verschieben und Skalieren von Fenstern. Es liest keine Fensterinhalte, protokolliert keine Aktionen und greift nicht auf das Netzwerk zu.

So erteilen Sie die Berechtigung

  1. Systemeinstellungen öffnen
  2. Zu Datenschutz & Sicherheit → Bedienungshilfen navigieren
  3. Workspace Container in der Liste finden und aktivieren
  4. Falls nicht aufgelistet, auf „+" klicken und manuell hinzufügen
  5. Workspace Container nach Erteilung der Berechtigung neu starten
⚠️
Sie müssen die App nach Erteilung der Berechtigung neu starten. Die macOS-Bedienungshilfen-Berechtigung wird nicht wirksam, während die App läuft — beenden und öffnen Sie Workspace Container erneut.
Einschränkungen

Bekannte Einschränkungen

!

Keine echte Fenstereinbettung

macOS erlaubt es nicht, das Fenster einer App in eine andere einzubetten. Workspace Container simuliert den Container-Effekt durch Steuerung von Fensterpositionen und -größen, anstatt echte Einbettung zu verwenden.

!

Einige Apps sind möglicherweise inkompatibel

Manche Apps können die externe Steuerung ihrer Fenster einschränken. Wenn ein App-Fenster nicht gebunden oder verschoben werden kann, liegt dies in der Regel an einer Einschränkung der jeweiligen App.

!

Vollbild-Apps

App-Fenster im nativen macOS-Vollbildmodus können nicht von Workspace Container gesteuert werden. Beenden Sie den Vollbildmodus vor dem Binden.

!

Mehrere Bildschirme

Die aktuelle Version ist primär für Einzelbildschirm-Setups optimiert. In Multi-Display-Umgebungen sollten Container und gebundene Fenster auf demselben Bildschirm sein.

FAQ

Häufig gestellte Fragen

Q Das Binden eines Fensters hat keine Wirkung?
Stellen Sie sicher, dass Sie die Bedienungshilfen-Berechtigung erteilt und Workspace Container danach neu gestartet haben. Prüfen Sie unter Systemeinstellungen → Datenschutz & Sicherheit → Bedienungshilfen.
Q Werden meine Apps geschlossen, wenn ich den Container schließe?
Nein. Das Schließen des Containers löst nur die Fensterbindungen. Alle App-Fenster bleiben an ihren aktuellen Positionen — sie werden weder geschlossen noch minimiert.
Q Sammelt Workspace Container meine Daten?
Nein. Workspace Container sammelt keinerlei Nutzerdaten, stellt keine Netzwerkanfragen und enthält keine Drittanbieter-SDKs. Ihre Privatsphäre ist vollständig geschützt. Weitere Details finden Sie in unserer Datenschutzerklärung.
Q Welche macOS-Versionen werden unterstützt?
Workspace Container unterstützt macOS 13.0 Ventura und neuer, einschließlich macOS 14 Sonoma und macOS 15 Sequoia. Sowohl Apple Silicon als auch Intel Macs werden unterstützt.
Q Kann ich eine Rückerstattung erhalten?
Workspace Container wird über den Mac App Store verkauft. Die Rückerstattungsrichtlinien folgen dem Standardverfahren von Apple. Um eine Rückerstattung zu beantragen, nutzen Sie Apples Seite „Problem melden".
