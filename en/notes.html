Permission

Accessibility Permission

🔐

Why is Accessibility permission needed?

Workspace Container uses the macOS Accessibility API (AXUIElement) to control the position and size of other app windows. This is the only legitimate way to manage windows on macOS — all window management apps (like Magnet, Rectangle) require this permission.

Workspace Container only uses this permission to move and resize windows. It does not read window content, log any actions, or access the network.

How to Grant Permission

  1. Open System Settings
  2. Go to Privacy & Security → Accessibility
  3. Find Workspace Container in the list and toggle it on
  4. If not listed, click "+" to add it manually
  5. Restart Workspace Container after granting permission
⚠️
You must restart the app after granting permission. macOS Accessibility permission does not take effect while the app is running — quit and reopen Workspace Container.
Limitations

Known Limitations

!

No True Window Embedding

macOS does not allow embedding one app's window inside another. Workspace Container simulates the container effect by controlling window positions and sizes, rather than true embedding.

!

Some Apps May Be Incompatible

A few apps may restrict external control of their windows. If an app window cannot be bound or moved, this is typically a limitation of that app itself.

!

Fullscreen Apps

App windows in macOS native fullscreen mode cannot be controlled by Workspace Container. Exit fullscreen mode before binding.

!

Multiple Displays

The current version is primarily optimized for single-display setups. In multi-display environments, the container and bound windows should be on the same screen.

FAQ

Frequently Asked Questions

Q Binding a window has no effect?
Make sure you've granted Accessibility permission and restarted Workspace Container afterward. Go to System Settings → Privacy & Security → Accessibility to check.
Q Will closing the container close my apps?
No. Closing the container only unbinds the windows. All app windows remain in their current positions — they won't be closed or minimized.
Q Does Workspace Container collect my data?
No. Workspace Container collects zero user data, makes no network requests, and includes no third-party SDKs. Your privacy is fully protected. See our Privacy Policy for details.
Q Which macOS versions are supported?
Workspace Container supports macOS 13.0 Ventura and later, including macOS 14 Sonoma and macOS 15 Sequoia. Both Apple Silicon and Intel Macs are supported.
Q Can I get a refund?
Workspace Container is sold through the Mac App Store. Refund policies follow Apple's standard process. To request a refund, submit through Apple's "Report a Problem" page.
