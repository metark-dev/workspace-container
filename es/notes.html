Permiso

Permiso de accesibilidad

🔐

¿Por qué se necesita el permiso de accesibilidad?

Workspace Container utiliza la API de accesibilidad de macOS (AXUIElement) para controlar la posición y el tamaño de las ventanas de otras aplicaciones. Esta es la única forma legítima de gestionar ventanas en macOS — todas las aplicaciones de gestión de ventanas (como Magnet, Rectangle) requieren este permiso.

Workspace Container solo utiliza este permiso para mover y redimensionar ventanas. No lee el contenido de las ventanas, no registra ninguna acción ni accede a la red.

Cómo conceder el permiso

  1. Abre Ajustes del Sistema
  2. Ve a Privacidad y seguridad → Accesibilidad
  3. Busca Workspace Container en la lista y actívalo
  4. Si no aparece en la lista, haz clic en "+" para añadirlo manualmente
  5. Reinicia Workspace Container después de conceder el permiso
⚠️
Debes reiniciar la aplicación después de conceder el permiso. El permiso de accesibilidad de macOS no surte efecto mientras la aplicación está en ejecución — cierra y vuelve a abrir Workspace Container.
Limitaciones

Limitaciones conocidas

!

Sin incrustación real de ventanas

macOS no permite incrustar la ventana de una aplicación dentro de otra. Workspace Container simula el efecto de contenedor controlando las posiciones y tamaños de las ventanas, en lugar de una incrustación real.

!

Algunas aplicaciones pueden ser incompatibles

Algunas aplicaciones pueden restringir el control externo de sus ventanas. Si una ventana de aplicación no se puede vincular o mover, esto suele ser una limitación de la propia aplicación.

!

Aplicaciones en pantalla completa

Las ventanas de aplicaciones en modo de pantalla completa nativo de macOS no pueden ser controladas por Workspace Container. Sal del modo de pantalla completa antes de vincular.

!

Múltiples pantallas

La versión actual está optimizada principalmente para configuraciones de una sola pantalla. En entornos con múltiples pantallas, el contenedor y las ventanas vinculadas deben estar en la misma pantalla.

FAQ

Preguntas frecuentes

Q ¿Vincular una ventana no tiene efecto?
Asegúrate de haber concedido el permiso de accesibilidad y de haber reiniciado Workspace Container después. Ve a Ajustes del Sistema → Privacidad y seguridad → Accesibilidad para verificar.
Q ¿Cerrar el contenedor cerrará mis aplicaciones?
No. Cerrar el contenedor solo desvincula las ventanas. Todas las ventanas de las aplicaciones permanecen en sus posiciones actuales — no se cerrarán ni se minimizarán.
Q ¿Workspace Container recopila mis datos?
No. Workspace Container no recopila ningún dato de usuario, no realiza solicitudes de red y no incluye SDKs de terceros. Tu privacidad está completamente protegida. Consulta nuestra Política de privacidad para más detalles.
Q ¿Qué versiones de macOS son compatibles?
Workspace Container es compatible con macOS 13.0 Ventura y posteriores, incluyendo macOS 14 Sonoma y macOS 15 Sequoia. Se admiten tanto Mac con Apple Silicon como con Intel.
Q ¿Puedo obtener un reembolso?
Workspace Container se vende a través de la Mac App Store. Las políticas de reembolso siguen el proceso estándar de Apple. Para solicitar un reembolso, envía tu solicitud a través de la página "Informar de un problema" de Apple.
Q ¿Cómo puedo obtener ayuda o reportar un problema?
