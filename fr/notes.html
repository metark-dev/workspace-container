Permission

Permission d'accessibilité

🔐

Pourquoi la permission d'accessibilité est-elle nécessaire ?

Workspace Container utilise l'API d'accessibilité de macOS (AXUIElement) pour contrôler la position et la taille des fenêtres d'autres applications. C'est le seul moyen légitime de gérer les fenêtres sur macOS — toutes les applications de gestion de fenêtres (comme Magnet, Rectangle) nécessitent cette permission.

Workspace Container utilise uniquement cette permission pour déplacer et redimensionner les fenêtres. Il ne lit pas le contenu des fenêtres, n'enregistre aucune action et n'accède pas au réseau.

Comment accorder la permission

  1. Ouvrez les Réglages Système
  2. Allez dans Confidentialité et sécurité → Accessibilité
  3. Trouvez Workspace Container dans la liste et activez-le
  4. S'il n'apparaît pas, cliquez sur « + » pour l'ajouter manuellement
  5. Redémarrez Workspace Container après avoir accordé la permission
⚠️
Vous devez redémarrer l'application après avoir accordé la permission. La permission d'accessibilité de macOS ne prend pas effet tant que l'application est en cours d'exécution — quittez et rouvrez Workspace Container.
Limitations

Limitations connues

!

Pas d'intégration réelle des fenêtres

macOS ne permet pas d'intégrer la fenêtre d'une application dans une autre. Workspace Container simule l'effet de conteneur en contrôlant les positions et tailles des fenêtres, plutôt qu'une véritable intégration.

!

Certaines applications peuvent être incompatibles

Quelques applications peuvent restreindre le contrôle externe de leurs fenêtres. Si une fenêtre d'application ne peut pas être liée ou déplacée, c'est généralement une limitation de cette application elle-même.

!

Applications en plein écran

Les fenêtres d'applications en mode plein écran natif de macOS ne peuvent pas être contrôlées par Workspace Container. Quittez le mode plein écran avant de lier.

!

Écrans multiples

La version actuelle est principalement optimisée pour les configurations à écran unique. Dans les environnements multi-écrans, le conteneur et les fenêtres liées doivent être sur le même écran.

FAQ

Questions fréquentes

Q La liaison d'une fenêtre n'a aucun effet ?
Assurez-vous d'avoir accordé la permission d'accessibilité et d'avoir redémarré Workspace Container ensuite. Allez dans Réglages Système → Confidentialité et sécurité → Accessibilité pour vérifier.
Q La fermeture du conteneur fermera-t-elle mes applications ?
Non. La fermeture du conteneur ne fait que délier les fenêtres. Toutes les fenêtres d'applications restent à leur position actuelle — elles ne seront ni fermées ni réduites.
Q Workspace Container collecte-t-il mes données ?
Non. Workspace Container ne collecte aucune donnée utilisateur, n'effectue aucune requête réseau et n'inclut aucun SDK tiers. Votre vie privée est entièrement protégée. Consultez notre Politique de confidentialité pour plus de détails.
Q Quelles versions de macOS sont prises en charge ?
Workspace Container prend en charge macOS 13.0 Ventura et ultérieur, y compris macOS 14 Sonoma et macOS 15 Sequoia. Les Mac Apple Silicon et Intel sont tous deux pris en charge.
Q Puis-je obtenir un remboursement ?
Workspace Container est vendu via le Mac App Store. Les politiques de remboursement suivent le processus standard d'Apple. Pour demander un remboursement, soumettez votre demande via la page « Signaler un problème » d'Apple.
Q Comment obtenir de l'aide ou signaler un problème ?
