접근성 권한이 필요한 이유

Workspace Container는 macOS 접근성 API(AXUIElement)를 사용하여 다른 앱 윈도우의 위치와 크기를 제어합니다. 이것은 macOS에서 윈도우를 관리하는 유일한 공식 방법이며, 모든 윈도우 관리 앱(Magnet, Rectangle 등)이 이 권한을 필요로 합니다.

Workspace Container는 이 권한을 윈도우 이동과 크기 조절에만 사용합니다. 윈도우 내용을 읽거나, 동작을 기록하거나, 네트워크에 접근하지 않습니다.

권한 허용 방법

  1. 시스템 설정 열기
  2. 개인정보 보호 및 보안 → 접근성으로 이동
  3. 목록에서 Workspace Container를 찾아 활성화
  4. 목록에 없으면 "+" 버튼을 클릭하여 수동으로 추가
  5. 권한 허용 후 Workspace Container 재시작
⚠️
권한 허용 후 반드시 앱을 재시작해야 합니다. macOS 접근성 권한은 앱 실행 중에는 적용되지 않습니다 — Workspace Container를 종료한 후 다시 실행하세요.
제한사항

알려진 제한사항

!

실제 윈도우 임베딩 불가

macOS는 한 앱의 윈도우를 다른 앱 안에 임베딩하는 것을 허용하지 않습니다. Workspace Container는 실제 임베딩이 아닌 윈도우 위치와 크기를 제어하여 컨테이너 효과를 시뮬레이션합니다.

!

일부 앱 호환 불가

일부 앱은 외부에서 윈도우를 제어하는 것을 제한할 수 있습니다. 앱 윈도우를 바인딩하거나 이동할 수 없는 경우, 이는 일반적으로 해당 앱 자체의 제한사항입니다.

!

전체 화면 앱

macOS 기본 전체 화면 모드의 앱 윈도우는 Workspace Container로 제어할 수 없습니다. 바인딩하기 전에 전체 화면 모드를 해제하세요.

!

다중 디스플레이

현재 버전은 주로 단일 디스플레이 환경에 최적화되어 있습니다. 다중 디스플레이 환경에서는 컨테이너와 바인딩된 윈도우가 같은 화면에 있어야 합니다.

FAQ

자주 묻는 질문

Q 윈도우를 바인딩해도 반응이 없나요?
접근성 권한을 허용했는지, 그리고 허용 후 Workspace Container를 재시작했는지 확인하세요. 시스템 설정 → 개인정보 보호 및 보안 → 접근성에서 확인할 수 있습니다.
Q 컨테이너를 닫으면 앱도 종료되나요?
아니요. 컨테이너를 닫으면 윈도우 바인딩만 해제됩니다. 모든 앱 윈도우는 현재 위치에 그대로 유지되며, 닫히거나 최소화되지 않습니다.
Q Workspace Container가 데이터를 수집하나요?
아니요. Workspace Container는 사용자 데이터를 일절 수집하지 않으며, 네트워크 요청을 하지 않고, 서드파티 SDK를 포함하지 않습니다. 개인정보가 완벽하게 보호됩니다. 자세한 내용은 개인정보 보호정책을 참조하세요.
Q 어떤 macOS 버전을 지원하나요?
Workspace Container는 macOS 13.0 Ventura 이상을 지원하며, macOS 14 Sonoma와 macOS 15 Sequoia도 포함됩니다. Apple Silicon과 Intel Mac 모두 지원합니다.
Q 환불받을 수 있나요?
Workspace Container는 Mac App Store를 통해 판매됩니다. 환불 정책은 Apple의 표준 절차를 따릅니다. 환불을 요청하려면 Apple의 "문제 신고" 페이지에서 제출하세요.
